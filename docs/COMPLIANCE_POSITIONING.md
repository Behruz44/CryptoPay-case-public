## 1. Узбекистан — домашний рынок

### Регуляторная база

- Регулятор: **НАПП** (Национальное агентство перспективных проектов) — [napp.uz](https://napp.uz)
- Ключевой акт: Положение о лицензировании провайдеров услуг в сфере оборота крипто-активов (рег. №3380 от 15.08.2022, изм. от 28.02.2026)
- С 01.01.2023 деятельность без лицензии НАПП **запрещена**

### Критическое ограничение

**Крипто-активы не являются средством платежа в Республике Узбекистан** (за исключением случаев, установленных законодательством). Это означает: прямая модель «оплата товаров USDT через QR» в текущем виде **не может быть запущена** в Узбекистане.

### Легальные пути для CryptoPay в Узбекистане

| Путь | Суть | Совпадение с нашим стеком |
|---|---|---|
| **Лицензия крипто-депозитария** | Хранение крипто-активов клиентов | ~70% — HD-кошельки, sweep, cold storage, signing-изоляция, KYC |
| **Лицензия крипто-магазина** | Покупка/продажа крипты за сумы | ~60% — конвертационный движок, KYC, фиатные расчёты |
| **White-label для лицензиатов** | Продажа/лицензирование технологии действующим лицензиатам | 100% — без собственной лицензии |
| **Конвертационная модель** | Крипта мгновенно конвертируется лицензиатом, мерчант получает сумы | Требует юридической проработки + партнёрства |

### Действующие лицензиаты (потенциальные покупатели/партнёры)

По публичному реестру НАПП: крипто-биржи (Coinpay, UzNEX), крипто-депозитарии (Paynet Crypto), крипто-магазины (Crypto Market, Crypto Express, Cryptocorp, Crypto Magnat, Crypto Trade Net). Актуальный реестр: napp.uz → Электронный реестр лицензий.

## 2. Альтернативные юрисдикции

| Юрисдикция | Регулятор | Плюсы | Минусы |
|---|---|---|---|
| **Казахстан (AIFC)** | AFSA | Ближайший рынок, русскоязычный, FinTech Lab (регуляторная песочница), англосаксонское право | Крипта работает в периметре AIFC |
| **Кыргызстан** | Финнадзор КР | Либеральное крипто-законодательство, низкая стоимость входа | Малый рынок |
| **Грузия** | НБ Грузии | Простая VASP-регистрация, крипто-friendly | Малый рынок, конкуренция |
| **ОАЭ (Дубай)** | VARA | Большой рынок, крипто-платежи развиваются | Лицензия $100k+, высокие требования |
| **ЕС** | MiCA/CASP | Огромный рынок | Долго (12+ мес), дорого, высокий комплаенс |

**Рекомендация по приоритету: 1) B2B в Узбекистане (white-label), 2) Казахстан AIFC, 3) Кыргызстан/Грузия как лёгкий вход.**

## 3. Стратегия выхода (по шагам)

### Фаза 3 — Сделка / запуск (3–6 месяцев)
- [ ] Вариант A: продажа технологии лицензиату (asset sale)
- [ ] Вариант B: white-label лицензия + техподдержка (recurring revenue)
- [ ] Вариант C: собственная лицензия депозитария (капиталоёмко, требует юрлица и уставного капитала)

## 4. Что потребует НАПП / покупатель при due diligence

1. **Чистота IP** — код написан одним автором, без заимствований под несовместимыми лицензиями (проверить зависимости: все npm-пакеты MIT/Apache)
2. **Security-аудит** — внутренний есть (см. SECURITY_AUDIT_REPORT.md); внешний потребуется перед запуском
3. **AML/KYC-соответствие** — Sumsub-интеграция покрывает идентификацию; AML-транзакционный мониторинг — доработка
4. **Данные** — локализация персональных данных (в Узбекистане — требование хранения в стране)
5. **Финмодель** — для лицензии: уставный капитал, источники средств учредителей
